# 성장주 목표주가 컨센서스 설계

## 목적

`/growth`의 `03 매수 검증` 화면에서 성장주 점수와 기술적 타이밍뿐 아니라 증권가 목표주가를 참고할 수 있게 한다. 목표주가는 자동 매수 신호가 아니라 참고 정보이며, 미국과 국내 종목에 같은 UI 계약을 제공한다.

## 범위

### 공통 표시 항목

- 현재가
- 평균 목표주가
- 중앙값 목표주가
- 최저 목표주가
- 최고 목표주가
- 평균 목표주가 기준 상승 여력
- 참여 기관 수
- 기준일
- 데이터 출처

기존 PEG, Forward PER, PSR, 컨센서스 성장률, EMA 타이밍 판정은 그대로 유지한다.

### 미국 종목

- 공급원: FMP `Price Target Consensus API`
- 조회 시점: 사용자가 `03 매수 검증`에서 종목을 선택할 때
- 캐시: 종목별 하루
- FMP가 제공하는 평균, 중앙값, 최저, 최고 목표주가를 사용한다.
- 참여 기관 수나 기준일을 FMP 응답이 제공하지 않으면 `null`로 반환한다.

### 국내 종목

- 공급원: KIS `국내주식 종목투자의견` API
- 조회 시점: 사용자가 `03 매수 검증`에서 종목을 선택할 때
- 캐시: 종목별 하루
- 최근 6개월 이내 리포트만 사용한다.
- 동일 증권사의 리포트가 여러 건이면 가장 최근 리포트만 사용한다.
- 유효한 목표주가를 가진 최신 리포트들로 평균, 중앙값, 최저, 최고, 참여 기관 수를 계산한다.
- 기준일은 집계에 포함된 최신 리포트 발표일이다.

## 데이터 흐름

신규 모듈 `core/data/price_target_consensus.py`가 공급원 차이를 숨긴다.

```text
GET /api/growth/ticker/{ticker}/valuation
  -> 기존 quant_scores 밸류에이션 조회
  -> price_target_consensus.fetch_price_target_consensus(ticker)
     -> 미국: FMP
     -> 국내: KIS
  -> 공통 응답으로 병합
  -> frontend valuation-card 표시
```

전 종목 배치에는 목표주가 수집을 추가하지 않는다. 사용자가 실제 검증하는 종목만 온디맨드로 조회해 외부 API 호출량과 배치 시간을 제한한다.

## 상태 모델

목표주가 조회 결과는 항상 공통 상태를 포함한다.

| 상태 | 의미 | UI 표시 | 캐시 정책 |
|---|---|---|---|
| `ok` | 목표주가 집계 성공 | 목표주가와 상승 여력 표시 | 종목별 하루 |
| `no_data` | 유효한 목표주가 없음 | `최근 6개월 내 목표주가 데이터 없음` | 종목별 하루 |
| `rate_limited` | FMP 또는 KIS 호출 제한 | 공급원별 한도 초과 안내 | 공급원 단위 오류 캐시 |
| `temporary_error` | 외부 API 일시 오류 | 잠시 후 재시도 안내 | 짧은 오류 캐시 |
| `missing_key` | API 키 미설정 | API 키 설정 필요 안내 | 재호출하지 않음 |

응답 필드:

- `consensus_status`
- `consensus_message`
- `consensus_source`
- `retry_after`
- `current_price`
- `target_mean`
- `target_median`
- `target_low`
- `target_high`
- `upside_pct`
- `analyst_count`
- `consensus_as_of`

## 한도 초과 처리

FMP 무료 플랜의 일일 호출 한도를 모두 사용한 뒤 미국 종목을 조회하면 `rate_limited`를 반환한다. PEG와 EMA 검증은 계속 표시하며 목표주가 영역만 한도 초과 안내를 보여준다.

한도 초과를 감지한 뒤에는 당일 다른 미국 종목을 열어도 FMP를 재호출하지 않는다. 외부 응답에서 정확한 재시도 가능 시각을 알 수 있으면 `retry_after`에 넣고, 알 수 없으면 `null`을 반환해 UI에서 `다음 한도 초기화 이후`라고 안내한다.

KIS 제한은 동일한 상태로 표현하되 짧은 오류 캐시를 사용한다. KIS는 일시적인 호출 제한일 수 있으므로 당일 전체 차단으로 확대하지 않는다.

## 오류 처리

- 외부 API 오류는 `/valuation` 자체를 실패시키지 않는다.
- 기존 밸류에이션 정보는 외부 목표주가 조회 실패와 무관하게 표시한다.
- 서버 로그에는 공급원과 실패 유형을 기록하되 API 키는 기록하지 않는다.
- 목표주가가 없다는 사실과 API 오류를 구분한다.

## 테스트

- FMP 정상 응답 파싱
- FMP 한도 초과 응답 감지 및 공급원 단위 차단
- 국내 최근 6개월 필터
- 국내 증권사별 최신 리포트 선택
- 국내 목표주가 집계
- 목표주가 없음 상태
- `/valuation` 공통 응답 계약
- 목표주가 조회 실패 시 기존 PEG 응답 유지
- UI lint와 브라우저 실제 시나리오 확인
